1. このアプリで何ができるか

text-viz は 日本語/英語の自由記述テキスト を多角的に解析する GUI です。 「どんな語が多いか」から「潜在的な話題は何か」「時期ごとにどう変わったか」まで、 計量テキスト分析(頻度・共起・対応分析・コーディング — KH Coder で知られる領域)と 現代的 NLP(分散表現・NER)を横断できます。

3 つの層で理解する

層問い対応する分析
記述統計どんな語が、どのくらい出るか?頻度、N-gram、TF-IDF、KWIC
関係性・潜在構造語 / 文書 / 時期の間にどんな関係があるか?共起、対応分析、クラスタリング、トピック
意味処理「意味」そのものを扱えるか?分散表現、類似検索、NER、係り受け、感情

2. 頻度分析 — TF / DF / TF-IDF

最も基礎的な「どの語が多いか」を 3 つの観点から捉える。

TF (Term Frequency, 単語頻度)
定義
語 w が全コーパスで出現する総回数
範囲
0 以上の整数
意味
「最も素朴な重要度」。ただし the, is, の, です 等が上位に来やすい
解釈
ドメイン固有語は強い手がかり / 汎用語はノイズ
使い方
まず表示して全体像を把握 → ストップワード(サイドバー)で汎用語を除外
DF (Document Frequency, 文書頻度)
定義
語 w が出現する文書数
範囲
0 ≤ DF ≤ 全文書数 N
意味
広く使われているかの指標(広いほど区別に不向き)
使い方
単体ではなく TF-IDF の計算基礎として重要
TF-IDF
TF-IDF(w, d) = TF(w, d) × log( N / DF(w) )
意味
「その文書で多い」かつ「他の文書では珍しい」語を高スコア
解釈
文書を特徴づける語。検索 / 文書分類 / 推薦の基礎
例
カルテ 100 件中 1 件だけに出る hypertension は高 TF-IDF、全件に出る 患者 はほぼ 0
ストップワードで汎用語を除去したら、TF と TF-IDF を並べて「順位が入れ替わる語」に注目すると、文書を特徴づける語が見つかります。

3. KWIC と N-gram

頻度だけでは見落とす「語の用法」「定型表現」を捉える。

N-gram

定義
N 個の連続する語
意味
複合語・定型表現を単位として扱う(例: bigram で quality of care)
使い方
単語単位では見えない言い回しを発見。N=2〜3 が定番、N=4+ は特定フレーズ検索向け

KWIC (Keyword In Context)

定義
指定キーワードの前後 W 語を切り取って並べたコンコーダンス
意味
「その語がどう使われているか」を文脈で確認
解釈
同じ語でも複数の意味・用法があり、頻度だけで判断すると誤解のもとに

4. 共起分析 — 5 つの指標

2 語の共起強度を測る指標は目的によって使い分ける。

共起の範囲 (scope)

範囲意味用途
document同じ文書内に出現論文 / レポート単位の話題関連性
sentence同じ文内文法的・意味的に近い関係
window窓幅 W 内局所的な連語・フレーズ

5 つの指標 — 何が違うか

Jaccard 集合の類似度
J(A, B) = |A ∩ B| / |A ∪ B|
範囲
[0, 1]
特徴
頻度の絶対値に依らない、稀語にも公平
用途
辞書学習、語彙の類似度比較
Dice Jaccard の別バリアント
D(A, B) = 2|A ∩ B| / (|A| + |B|)
特徴
Jaccard より共起側を強調(2 倍)
使い分け
Jaccard と似るが、境界例で少し異なる。どちらでも実用上大差なし
PMI 相互情報量 — 意外性
PMI(A, B) = log( P(A, B) / ( P(A) × P(B) ) )
範囲
(-∞, +∞)、独立なら 0
意味
「たまたまの共起」より どれだけ頻繁か。大 = 意外性大
注意
稀語で過大評価される(分母が小さいため)
用途
専門用語 / 複合語発見、珍しい組み合わせ
t-score 統計的有意性
t = (c - expected) / √c
意味
期待共起数との差を標準化
特徴
PMI と異なり稀語に過剰反応しない
用途
統計的に意味ある共起を選別
対数尤度比 (Dunning) 科学論文で標準
意味
観測した共起が独立仮説と比べて何倍「起こりやすい」か
特徴
χ² に似るがサンプルサイズに対してより頑健
用途
コーパス言語学、専門用語抽出
迷ったら: まず Jaccard(安定)。珍しい組み合わせ発見は PMI。厳密な有意性は log-likelihood。

5. 共起ネットワーク — 構造と中心性

共起行列を視覚化してコーパスの「話題の地図」を描く。

ハブ(媒介中心性が高い)

コミュニティ検出

手法特徴用途
LouvainModularity 最大化、高速、標準迷ったらこれ
Greedyシンプル、大規模でも軽量数千ノード超のグラフ
Girvan-Newmanエッジ媒介性で逐次除去、精密だが遅い小規模・厳密性重視

中心性指標 — どのノードが重要か

指標意味高いノードの特徴
Degree直接の隣接数「いろいろな語と共起する」
Betweenness (媒介)他ノード間の最短経路が何度通るか橋渡し役、分野横断ハブ
Closeness (近接)他ノードへの平均距離の逆数情報が早く届く中心語
Eigenvector重要な隣人を持つほど重要影響力のあるノード
PageRankEigenvector の改良版(Google 検索)堅牢な影響力指標
医療安全レポートで「患者」「症状」等の汎用語は Degree が高くても情報量が少ない。ストップワードに追加するか、Betweenness を見て「橋渡し語」に絞ると含蓄が得られる。

6. 対応分析 (CA) — カテゴリと語を同じ地図に

計量テキスト分析の代表的な手法(KH Coder でも実装されている)。カテゴリ変数 × 頻出語 のクロス表を 2 次元平面に布置し、 「どのカテゴリにどの語が特徴的か」を一望できる。

近いほど関連性が強い。 原点から遠い語 / カテゴリはその分特徴的。

使い方

  1. サイドバーで「カテゴリ列」を選択(例: 部署、年度、重症度)
  2. 「対象上位語数」を 30〜50 に設定(少ないほど見やすい)
  3. 実行 → 布置図を確認
  4. 「語とカテゴリが重なる位置」がそのカテゴリの代表語
原点付近の語やカテゴリは「どこにも特徴的でない = 汎用的」。目印は遠い点。

7. クラスタリング + 次元削減

文書を自動的にグループ化し、2 次元空間で可視化する。

クラスタリング手法

手法仮定するクラスタ形状長所短所
KMeans球状高速、クラスタ数 k を指定非球状に弱い
階層 (Ward)任意デンドログラムで関係性が見える大規模で遅い
DBSCAN密度ノイズを自動除外、k 不要密度パラメータ調整が要る

シルエットスコア — クラスタの質

s(i) = (b(i) − a(i)) / max(a(i), b(i))
値解釈
+1 に近い良好 クラスタ内が密・クラスタ間が疎
0 付近境界 他のクラスタと区別困難
負悪い 誤クラスタ化の可能性

次元削減手法 — 2D プロット用

手法タイプ保存する構造使いどころ
PCA線形全体分散高速、初期確認
t-SNE非線形局所近傍クラスタ可視化、数千件まで
UMAP非線形局所+大域t-SNE より速く、10 万件級でも OK
MDS距離保存距離行列距離を直接解釈したい場合

8. 自己組織化マップ (SOM)

Kohonen (1982) が提案したニューラルネットワーク型の可視化手法。 高次元の文書ベクトルを **2 次元格子(マップ)上に配置**して、 似た文書が**近くのセルに集まる**ように学習する。

仕組みを 1 枚で

近いセルに、似ていない文書は離れたセルに配置される。 セル間の色が暗い(灰色)部分はクラスタ境界 — U-Matrix 可視化の原理。 → クラスタ数を指定せず自動で発見でき、同時に位相関係も保存できるのが強み。

学習アルゴリズム(概要)

  1. 初期化: 格子 (x × y) の各セルに重みベクトルをランダム配置
  2. 勝者選定 (BMU): 入力ベクトルに最も近いセル(Best Matching Unit)を選ぶ
  3. 重み更新: BMU と近傍のセルの重みを入力側に引き寄せる
  4. これを全入力・全反復で繰り返すと「近い入力は近いセルに集まる」マップが完成

U-Matrix の読み方

text-viz が出力する U-Matrix(Unified Distance Matrix)は**近傍セル間の距離**をヒートマップ化したもの。

セルの色意味解釈
明るい(白〜薄灰) 近傍と距離が小さい 同じクラスタ内 — 連続した話題
暗い(濃灰〜黒) 近傍と距離が大きい クラスタ境界 — 話題の切れ目
暗い線が格子を分断するように走っていたら、そこで話題が切り替わっている。 明るい領域の「面積」がクラスタの規模の目安になる。

text-viz での使い方

パラメータ
X × Y: 格子のサイズ (既定 10×10)。データ件数の √N × √N が目安。
iterations: 学習反復数 (既定 500)。少ないと収束不足、多すぎると過学習傾向。
入力
自動的に TF-IDF ベクトル化した文書行列を使用
出力
U-Matrix ヒートマップ(matplotlib)

他のクラスタリング手法との比較

観点KMeans階層UMAPSOM
クラスタ数要指定 (k)不要不要不要(格子サイズで決定)
トポロジー保存×△○◎
非線形構造×△◎◎
解釈性◎(重心)◎(樹形)△○(格子 + U-Matrix)
外れ値弱い弱い強い強い(孤立セルで検出)
大規模◎×◎△(格子サイズに依存)
SOM が向いている場面:
  • クラスタ数が事前に分からない
  • 話題の「連続性・位相」を見たい(近い話題が近くに配置される)
  • 数百〜数千文書をヒートマップ的に俯瞰したい
  • 外れ値(孤立した独特な文書)を目視で見つけたい

制限と注意点

  • 初期化依存性: 乱数シードで結果が変わる。安定させたい場合は複数回試行
  • 格子サイズの選択: 小さすぎるとセルに文書が詰め込まれ、大きすぎると空セルが多く荒くなる
  • 計算コスト: 反復数 × データ件数 × 格子サイズ に比例。1 万件 × 20×20 で数分
  • 次元の呪い: 超高次元(1 万次元超)では事前に PCA/UMAP で次元削減すると良い

9. 特徴語抽出 — グループ別の代表語

カテゴリ変数ごとに「そのグループで特に多い語」を統計的に抽出する。

指標理論特徴
χ² (カイ二乗) 期待頻度 vs 観測頻度の差の二乗和 直感的、標準的。サンプルサイズに敏感
対数尤度比 Dunning の対数尤度 χ² より頑健、稀語にも対応
Jaccard 文書レベルの Jaccard 解釈しやすい比率、頻度重視
「2023 年度 vs 2024 年度」や「ICU vs 内科病棟」など、比較したい切り口をカテゴリ列に設定すると、本当に差があるかが数値で把握できる。

10. トピックモデル — LDA / NMF

各文書の潜在的な「話題(トピック)」を教師なしで抽出する。

LDA のパラメータ

num_topics
5〜20 が実用範囲。多すぎると分散、少なすぎると粗い
passes
学習反復数。既定 5、精度を上げたいなら 10〜20
最適探索
コヒーレンス c_v で自動選択

コヒーレンス c_v

値解釈
≥ 0.5良好 トピックが人間に解釈しやすい
0.3〜0.5普通 やや曖昧
< 0.3悪い トピックが不明瞭

pyLDAvis の読み方

  • 円の大きさ: トピックのコーパス全体での出現割合
  • 円の距離: トピック間の類似度(PCoA で 2D 化)
  • 赤い棒: そのトピック内での語の出現頻度
  • 青い棒: コーパス全体での語の頻度
  • λ スライダー: トピック特異性 (0) か全体頻度 (1) か

NMF との使い分け

  • LDA: 確率的、文書がトピックの混合として解釈される
  • NMF: 代数的、高速、スパース。明確なトピックを得たい時

12. 類似検索 — FAISS

自由文クエリから意味的に近い文書を瞬時に検索する。

コサイン類似度

cos(A, B) = (A · B) / (|A| × |B|)
値意味
1.0完全に同じ方向(同義)
0.7〜1.0強く類似
0.3〜0.7ある程度関連
< 0.3関連薄い

タイプ検索精度速度メモリ適する規模

13. 固有表現抽出 (NER)

テキストから人名・組織・場所・日付などの「固有名詞 / 特定の表現」を自動で抽出する。

GiNZA の主要ラベル

ラベル意味例
PERSON人名山田太郎
ORG組織東京大学、JAXA
LOC / GPE場所東京、アメリカ
DATE日付2024年5月、令和6年
MONEY金額1000円
PRODUCT製品iPhone

医療 NER (MedNER-J)

病名・薬剤・症状など医療ドメインに特化。未導入時はルールベースにフォールバックします(同梱の医療同義語辞書を参照)。

大量文書の「実体統計」を取りたい時に便利。例: ORG ラベルで「どの機関が頻出か」、DATE で「どの年代が話題か」。

14. 係り受け解析

語と語の文法的な関係を抽出し、述語項構造(「何を」「どうする」「誰が」)を得る。

(投与, 薬)、subject_verb = (医師, 投与)

抽出可能な関係

関係例応用
verb_obj「薬を投与する」行為 — 対象の対抽出
subject_verb「患者が改善する」主体 — 行為の対抽出
adj_noun「重篤な副作用」修飾 — 被修飾の対抽出
単語の共起より厳密。「投与 と 薬」が単に一緒に出るのではなく、文法的に関係していることを確認できる。

15. 感情分析

文書のポジティブ / ネガティブ傾向を定量化する。

手法の使い分け

手法対象言語原理出力スコア
oseti日本語日本語評価極性辞書 + ルール(pos - neg) / (pos + neg)
VADER英語ソーシャル向け辞書 + ルールcompound [-1, 1]
custom_dict任意ユーザー定義の語 × スコア(pos - neg) / (pos + neg)

スコアの読み方

  • +0.5 以上: 明確にポジティブ
  • +0.2〜+0.5: やや肯定的
  • -0.2〜+0.2: 中立または評価未言及
  • -0.5〜-0.2: やや否定的
  • -0.5 以下: 明確にネガティブ
辞書ベースは皮肉・反語を検出できません。「薬が素晴らしく効かなかった」のような表現は誤判定します。重要な判断には KWIC で文脈確認を。

16. 時系列分析

日付列がある場合、テキストの「時間的変化」を追える。

単語トレンド

急増語検出

ratio = (比較期間 count + 1) / (基準期間 count + 1)
  • Laplace 平滑化 (+1) で 0 除算を回避
  • ratio が大きい語 = 新たに注目された語
  • 既存の頻出語で ratio ≈ 1 なら「ずっと話題」、0 付近なら「廃れた語」

期間別ネットワーク

時期ごとに別々の共起ネットワークを構築すると、コミュニティ構造の変化を追える(例: 「COVID」が孤立ノードだった時期 → 他のノードと統合された時期)。

17. コーディング

ユーザー定義のルールに従って文書にコードを付与し、集計する。 計量テキスト分析で広く用いられる手法で、KH Coder の「コーディングルール」機能と同種の仕組み。

ルールの書き方

透析関連:
  - 透析
  - HD
  - シャント
合併症:
  - 心不全
  - 糖尿病
  - 感染症
薬剤:
  - 抗菌薬
  - インスリン

得られる出力

  • コード別頻度: どのコードが最も出現するか
  • 文書 × コード行列: 各文書にどのコードが何回出現するか
  • コード × カテゴリのクロス集計: 部署別のコード出現数など
  • コード間共起ネットワーク: 上位概念の発見(例:「透析関連」と「合併症」が常に一緒に現れる)
定性研究で使うコードブック(概念の階層体系)をそのまま YAML にすると、定量集計が自動化できる。
